# 笔画分类问题解决方案

## 问题背景

项目需要将 Make Me a Hanzi 数据集中的每个汉字的每一笔归类到 29 种笔画类别（横、竖、撇、捺、点、提、竖钩、弯钩、斜钩、卧钩、横钩、横折、竖折、撇折、横折钩、横撇、横撇弯钩、横折弯钩、竖折折钩、横折折撇、竖提、撇点、横折提、横折弯、竖弯、竖弯钩、横斜钩、横折折折钩、横折折）。

## 原始方案的问题

最初的 `classify_stroke_by_svg()` 函数使用 SVG 路径（`strokes` 字段）进行分类。SVG 路径描述的是**笔画的填充轮廓**（封闭形状），而非实际的书写轨迹。这导致：

1. **SVG 轮廓是闭合形状**：例如一横的 SVG 可能是 `M 0 0 L 100 0 L 100 10 L 0 10 Z`，是一个矩形框，而非一条从左到右的线
2. **方向信息丢失**：从闭合轮廓中难以提取书写方向（起笔→收笔的方向）
3. **转折点检测不准确**：轮廓的四个角会被误判为多次方向变化
4. **分类结果极差**：几乎所有字符的笔画都被分到同一个错误类别（如 ID 18），因为分类器无法从轮廓中提取有意义的几何特征

## 解决方案

### 核心改动：从 SVG 轮廓切换到 median 中线数据

Make Me a Hanzi 的 `graphics.txt` 中每条记录除了 `strokes`（SVG 路径），还包含 `medians` 字段——这是**笔画的中心线轨迹**，由一系列 `[x, y]` 坐标点组成，直接表示书写时的运笔路径。

### 具体修改

#### 1. 新增 `parse_graphics_with_medians()` 函数（`parser.py:60-81`）

解析 `graphics.txt` 中的 `medians` 字段，返回 `{字符: [笔画中线列表]}`。每条中线是一个 `[[x1,y1], [x2,y2], ...]` 的坐标序列。

#### 2. 重写分类器为 `classify_stroke_by_median()`（`parser.py:176-387`）

基于 median 中线数据的几何特征进行分类，核心逻辑：

**特征提取：**
- **路径长度** (`path_len`)：所有线段长度之和
- **直线度** (`straightness`)：首尾直线距离 / 路径长度，越接近 1 越直
- **整体方向角** (`overall_angle`)：起点到终点的角度
- **初始/末尾段方向** (`init_angle`, `final_angle`)：第一段和最后一段的方向
- **统一转折检测**：遍历所有相邻线段，计算夹角
  - 夹角 > 15° 才算一次转折（过滤噪声）
  - 夹角 > 60° 归类为"折"（fold），≤ 60° 归类为"钩"（hook）
  - 记录每次转折的方向（左转/右转，通过叉积判断）

**分类决策树（按优先级）：**

1. **路径极短** (< 20 单位) → 点 (4)
2. **无转折 + 短促右上斜** → 提 (5)
3. **无转折 + 高直线度** → 根据角度分为 横(0)/竖(1)/撇(2)/捺(3)
4. **1 个转折**：
   - 大角度（折）：根据初始方向分为 横折(11)/竖折(12)/撇折(13)/横撇(15)/竖提(20)
   - 小角度（钩）：根据初始方向分为 横钩(10)/竖钩(6)/斜钩(8)/竖提(20)
5. **2 个转折**：根据折/钩的组合和初始方向，分为 横折钩(14)/横折弯钩(17)/竖折折钩(18)/横撇弯钩(16)/横折折撇(19)/横折折(28)/竖弯(24)/撇点(21)/横斜钩(26)/竖弯钩(25)/弯钩(7) 等
6. **3+ 个转折**：横折折折钩(27) 或 横折折撇(19)/横折折(28)
7. **全弧线**（所有转折都是小角度）：卧钩(9)/竖弯钩(25)/弯钩(7)
8. **Fallback**：根据初始方向和是否有钩进行兜底分类

#### 3. 更新 `build_taxonomy()` 函数（`parser.py:390-438`）

将 `parse_graphics()` 替换为 `parse_graphics_with_medians()`，将 `classify_stroke_by_svg()` 替换为 `classify_stroke_by_median()`。

#### 4. 更新测试用例（`tests/test_parser.py`）

- 新增 `TestParseGraphicsWithMedians` 测试类
- 将 `TestClassifyStroke` 替换为 `TestClassifyStrokeByMedian`，包含 24 个测试用例覆盖所有 29 种笔画类别
- 测试数据从 SVG 路径改为 median 坐标序列

### 关键设计决策

- **折与钩的区分阈值设为 60°**：这是书法中的关键区分——折是笔画的"骨架转折"（大角度），钩是笔画的"收笔小勾"（小角度）
- **转折检测阈值设为 15°**：过滤 median 数据中的微小抖动，避免将噪声误判为转折
- **提(5) 的检测放在直线笔画之前**：因为提和撇/捺在几何特征上相似（都是无转折斜线），但提更短且方向固定为右上

## 结果

重新运行 `build_taxonomy()` 后，`stroke_taxonomy.json` 中的 `char_to_strokes` 映射从几乎全部是 ID 18 变为多样化的笔画 ID 分布，不同字符的不同笔画被分到不同的合理类别中。
