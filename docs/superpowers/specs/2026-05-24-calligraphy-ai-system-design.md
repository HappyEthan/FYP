# 多模态融合 AI 书法教学与评估系统 — 需求规格说明书

## 1. 项目概述

### 1.1 项目背景

本项目是一个多模态融合的 AI 书法教学与评估系统。核心硬件为一支集成了软性触觉传感器笔尖和顶部视觉采集模块的智能毛笔。系统通过"视觉自动标注 + 触觉数据采集 + Transformer 时序建模"的技术路线，实现仅依靠笔尖触觉信号就能精准识别书写笔画、评估书法质量的目标。

### 1.2 目标用户

- **主要用户**：书法课堂教学场景（教师 + 多名学生使用），提供书写质量自动评分与反馈
- **次要用户**：研究实验平台（供研究人员采集数据、训练模型、验证算法）

### 1.3 项目范围

**本期范围（FYP 交付）：**
- 书法评分系统（笔画级 + 整字级）
- 笔迹还原可视化（2D 轨迹 + 压力热力图）
- 数据采集与模型训练全流程

**未来扩展（不在本期范围）：**
- VLA 模型笔迹复现
- IMU 传感器集成
- 实时推理模式

---

## 2. 系统整体架构

系统采用"硬件采集层 — 数据处理层 — 模型训练层 — 应用层"四层架构：

```
硬件采集层：智能毛笔（软性触觉传感器笔尖 + 顶部相机）
    ↓ USB 串口通信
数据处理层：视觉标注 → 标签生成 → 触觉-视觉对齐
    ↓
模型训练层：Transformer 时序模型训练 → 笔画识别 + 质量评估
    ↓
应用层：笔画识别、书写质量反馈、笔迹还原可视化
```

## 3. 核心硬件组成

### 3.1 软性触觉传感器笔尖

| 属性 | 说明 |
|------|------|
| **功能** | 实时采集书写过程中笔尖与纸张接触产生的多维触觉信号 |
| **采集数据** | 笔尖压力（力度）、笔尖运动方向（2D 向量） |
| **通信方式** | USB 串口与电脑通信，采样率 100-200Hz |
| **特点** | 软性传感器模拟真实毛笔弹性和触感，不影响正常书写体验 |

**传感器数据维度（每个时间步 3 维）：**

| 通道 | 符号 | 含义 | 值域 |
|------|------|------|------|
| 力度 | `f` | 笔尖压力标量 | [0, 1] |
| 方向 X | `dx` | 笔尖运动方向 X 分量 | [-1, 1] |
| 方向 Y | `dy` | 笔尖运动方向 Y 分量 | [-1, 1] |

> **注**：硬件具体型号和规格待后续补充。方向采用 2D 向量而非角度值，避免角度在 0°/360° 边界的不连续性，更利于神经网络学习。当前假设为 2 通道（力度 + 方向），后续若传感器规格变化可扩展。

### 3.2 顶部相机

| 属性 | 说明 |
|------|------|
| **功能** | 书写完成后拍摄最终成字照片，用于视觉标注和字形轮廓提取 |
| **安装位置** | 书写台上方固定，拍摄范围覆盖整个书写区域 |
| **采集数据** | 高清静态照片（非视频流） |
| **触发方式** | 书写完成后手动或自动触发拍摄 |

## 4. 核心功能详解

### 4.1 功能 1：视觉方案自动生成训练标签

**解决的问题**：触觉数据的人工标注成本极高，且难以精确到笔画级别。

**实现流程**：

1. 用户书写完成后，顶部相机拍摄最终成字照片
2. 使用 OCR 技术（推荐 PaddleOCR）识别字形，得到字标签（如"永"）
3. 根据字标签查询笔画类别库，获取该字的标准笔画序列（如"永" → [点, 横折钩, 横撇, 撇, 捺]）
4. 以标准笔画序列为先验知识，使用笔画分割算法将字拆分为独立笔画，得到每个笔画的字形轮廓
5. 使用视觉质量评估方法，对比用户字照片与标准字帖，自动生成字形质量标签（形态分、位置分、间架结构分）
6. 自动生成的标签作为模型训练的 Ground Truth

**优势**：自动化标注，降低数据采集成本，保证标签一致性和准确性。

#### 4.1.1 OCR 字识别

OCR 在系统中的角色是**标注阶段的辅助工具**，不参与实时推理。

| 属性 | 说明 |
|------|------|
| **使用阶段** | 仅数据标注阶段 |
| **推荐方案** | PaddleOCR（中文手写体识别 SOTA，支持离线部署） |
| **输入** | 顶部相机拍摄的成字照片 |
| **输出** | 字标签（如"永"） |
| **下游用途** | 查询笔画类别库，获取该字的标准笔画序列，作为笔画分割的先验知识 |

> 备选方案：Tesseract + 中文语言包（开源免费，但中文手写体准确率一般）、TrOCR（Transformer-based，需微调）。

#### 4.1.2 视觉质量评估

利用成字照片自动生成字形质量标签，作为模型训练的质量 Ground Truth。以下三种方案按推荐优先级排列：

**方案 A：传统图像处理（推荐 FYP 阶段使用）**

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1. 预处理 | 灰度化 + 二值化 + 去噪 | 将照片转为干净的二值图像 |
| 2. 边缘检测 | Canny / Sobel | 提取字形边缘轮廓 |
| 3. 笔画分割 | 基于标准笔画序列先验的骨架提取 + 笔画拆分 | 利用笔画类别库提供的笔画数量/类型约束 |
| 4. 形状匹配 | Chamfer Distance / DTW / SSIM | 逐笔画与标准字帖对比 |
| 5. 几何特征提取 | 笔画长度比、角度偏差、重心偏移、对称性 | 计算结构层面的偏差 |
| 6. 质量分映射 | 将距离/偏差值映射到 [0, 1] 区间 | 输出形态分、位置分、间架结构分 |

- **优势**：无需训练，可解释性强，FYP 阶段可快速跑通
- **劣势**：对草书、行书等连笔字体鲁棒性差

**方案 B：深度学习 Siamese Network**

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1. 特征提取 | 预训练 CNN（如 ResNet-18） | 分别提取用户字和标准字帖的视觉特征 |
| 2. 特征对比 | Siamese Network | 计算两个特征向量的嵌入距离 |
| 3. 质量回归 | MLP 回归头 | 将嵌入距离映射为多维度质量分 |

- **优势**：端到端，泛化能力强，可处理不同字体风格
- **劣势**：需要训练数据，可解释性弱

**方案 C：混合方案**

结合方案 A 的几何特征提取 + 方案 B 的轻量学习模型：
- 先用传统方法提取几何特征向量（笔画长度比、角度偏差、重心偏移、对称性等）
- 再用轻量 MLP 融合多特征，输出多维度质量分

- **优势**：兼顾可解释性和灵活性
- **劣势**：管线较复杂

> **推荐策略**：FYP 阶段先用方案 A 跑通全流程，方案 B/C 作为论文中"可扩展方向"讨论。

### 4.1A 笔画类别库

笔画类别库是系统的核心基础数据，为视觉标注提供笔画分类体系和字-笔画映射关系。

#### 数据来源

| 来源 | 内容 | 用途 |
|------|------|------|
| Make Me a Hanzi — `graphics.txt` | ~9000 汉字的笔画拆解 + SVG 路径数据 | 提取每个汉字的笔画序列和笔画形状 |
| Make Me a Hanzi — `dictionary.txt` | 汉字 → 笔画序列映射 | 建立字到笔画的查询索引 |
| 《现代汉语通用字笔顺规范》 | 国家语委发布的 7000 汉字规范笔顺 | 校验笔画序列的准确性 |
| 书法教材（楷书入门等） | 基本笔画 + 复合笔画的分类和写法 | 定义笔画类别体系 |

#### 笔画类别体系

| 大类 | 具体笔画 | 数量 |
|------|---------|------|
| 基本笔画 | 横、竖、撇、捺、点、提 | 6 |
| 钩类 | 竖钩、弯钩、斜钩、卧钩、横钩 | 5 |
| 折类 | 横折、竖折、撇折 | 3 |
| 复合笔画 | 横折钩、横撇、横撇弯钩、横折弯钩、竖折折钩、横折折撇、竖提、撇点、横折提、横折弯、竖弯、竖弯钩、横斜钩、横折折折钩 等 | ~15 |
| **合计** | | **~29 类** |

#### 存储格式

笔画类别库存放在 `data/stroke_taxonomy/` 目录下，JSON 格式：

```json
{
  "strokes": [
    {
      "id": 0,
      "name": "横",
      "name_pinyin": "heng",
      "type": "basic",
      "svg_path": "M 0,0 L 100,0",
      "description": "从左向右的水平笔画"
    },
    {
      "id": 14,
      "name": "横折钩",
      "name_pinyin": "hengzhegou",
      "type": "compound",
      "svg_path": "M 0,0 L 80,0 L 80,60 L 75,55",
```

#### 使用流程

```
OCR 识别出字标签 "永"
    ↓
查 char_to_strokes["永"] → [3, 14, 1, 4, 5]
    ↓
查 strokes[3] → "点", strokes[14] → "横折钩", ...
    ↓
得到该字的标准笔画序列 → 作为笔画分割的先验约束
```

### 4.2 功能 2：触觉数据与视觉标签的时序对齐

**解决的问题**：触觉数据和视觉标签来自不同传感器，需要精确对齐才能构建训练数据集。

**实现流程**：

1. 书写过程中，触觉传感器持续采集数据流
2. 利用"抬笔/落笔"事件（触觉传感器压力归零/突增）将连续数据流自动分割为 N 个笔画片段
3. 笔画片段按书写时间顺序排列
4. 视觉笔画分割结果按书写顺序与触觉片段一一匹配对应
5. 建立"触觉数据片段 — 笔画类别 — 字形轮廓"的一一对应关系，形成完整训练数据集

**关键技术**：基于触觉压力阈值的笔画分割算法，确保抬笔/落笔事件检测准确。

### 4.3 功能 3：基于 Transformer 的笔画识别与书写质量评估

**解决的问题**：利用对齐好的触觉-标签数据，训练一个仅依靠触觉信号就能识别笔画和评估书写质量的模型。

#### 4.3.1 数据预处理

- 触觉数据归一化、去噪、补全
- 每个笔画的触觉片段转换为固定长度时序序列（L=128，短则 padding 长则截断）
- 质量标签量化

#### 4.3.2 模型输入向量设计

**传感器原始数据（每个时间步 3 维）：**

| 通道 | 符号 | 含义 | 值域 |
|------|------|------|------|
| 力度 | `f` | 笔尖压力标量 | [0, 1] |
| 方向 X | `dx` | 笔尖运动方向 X 分量 | [-1, 1] |
| 方向 Y | `dy` | 笔尖运动方向 Y 分量 | [-1, 1] |

**输入向量构造流程：**

```
原始输入 raw_t = [f, dx, dy] ∈ R³

Step 1 — 传感器嵌入:
    tactile_emb = Linear(3, 64) → GELU → LayerNorm → R^64

Step 2 — 位置编码:
    pos_enc = SinusoidalPositionEncoding(t, d=64) ∈ R^64

Step 3 — 片段全局特征 (对所有时间步 t 相同):
    global_feat = concat([
        mean(f), std(f), max(f), min(f),      # 力度统计
        mean(|dx|), mean(|dy|),                # 方向统计
        trajectory_length,                      # 轨迹长度
        duration                                # 书写时长
    ]) ∈ R^8
    → Linear(8, 32) → R^32

Step 4 — 拼接 + 投影:
    x_t = concat([tactile_emb, pos_enc, global_feat]) ∈ R^160
    → Linear(160, d_model) → R^d_model
```

**序列级输入：** 在笔画片段开头添加 `[CLS]` token，其最终编码用于下游任务（分类 + 回归）。

**笔画标签结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `stroke_type` | int (0~28) | 笔画类别 ID，对应笔画类别库中的分类 |
| `quality_shape` | float [0,1] | 笔画形态质量（视觉方案生成） |
| `quality_force` | float [0,1] | 笔画力度质量（触觉方案生成） |
| `quality_speed` | float [0,1] | 笔画速度质量（触觉方案生成） |
| `quality_position` | float [0,1] | 笔画位置质量（视觉方案生成） |

#### 4.3.3 质量标签定义

**笔画级质量标签（多维度）：**

| 维度 | 说明 | 评分基准 | 输出形式 |
|------|------|----------|----------|
| 笔画形态 | 笔画形状与标准笔画库的相似度 | 视觉：对比标准字帖笔画轮廓 | 连续值 0-1 |
| 笔画力度 | 压力分布是否均匀，起笔收笔力度变化是否到位 | 触觉：对比标准笔画库触觉数据 | 连续值 0-1 |
| 笔画速度 | 书写速度是否平稳 | 触觉：对比标准笔画库触觉数据 | 连续值 0-1 |
| 笔画位置 | 该笔画在字中的相对位置是否准确 | 视觉：对比标准字帖 | 连续值 0-1 |

**整字级质量标签（多维度）：**

| 维度 | 说明 | 评分基准 | 输出形式 |
|------|------|----------|----------|
| 间架结构 | 各笔画空间关系是否协调（重心、比例、对称） | 视觉：对比标准字帖 | 连续值 0-1 |
| 整体形态 | 整字与标准字帖的整体相似度 | 视觉：对比标准字帖 | 连续值 0-1 |

**标签形式**：多维度独立评分 + 加权综合分。

#### 4.3.4 标准笔画库

- 邀请书法专家使用智能毛笔书写，采集其触觉数据作为"满分标准"
- 每个笔画类别（横、竖、撇、捺等）存储多条专家触觉数据作为参考基准
- 评分时，将用户触觉数据与标准笔画库中的同类别数据进行相似度对比

#### 4.3.5 模型架构

- **核心模型**：Transformer 编码器（具体变体待探讨）
- **输入层**：多维触觉信号 → 嵌入向量（详见 4.3.2 输入向量设计）
- **编码器层**：多头自注意力机制捕捉触觉数据中的长距离依赖关系
- **输出层**（双分支）：
  - 笔画分类分支：`Linear(d_model, 29)` + Softmax → 29 类笔画类别
  - 质量评估分支：`Linear(d_model, 4)` + Sigmoid → 4 维质量分

#### 4.3.6 模型训练

- 使用对齐好的"触觉数据 — 笔画标签 — 质量标签"数据集进行监督训练
- 多任务损失函数：笔画分类损失 + 质量评估损失（各维度独立损失 + 综合分损失）
- 验证集调参，防止过拟合

#### 4.3.7 Active Learning（方法论层面）

- 自动规则生成弱标签 → 训练初始模型
- 模型输出预测 + 置信度
- 选出低置信度样本，交由人工（书法专家）复核
- 修正标签后重新训练，循环迭代
- 作为论文方法论章节呈现，实际系统先用自动规则跑通

### 4.4 功能 4：笔迹还原可视化

**解决的问题**：书写完成后，在屏幕上生成书写笔迹还原动画，让观众清楚看到每个笔画的书写细节。

**实现流程**：

1. 触觉数据提供局部运动信息（方向、压力、速度随时间变化）
2. 相机照片提取字形轮廓，提供全局空间约束（每个笔画的起止位置和轮廓边界）
3. 将触觉时序信息映射到视觉笔画轮廓上，生成带时序的笔迹动画
4. 叠加压力热力图（压力大处颜色深，压力小处颜色浅）

**可视化内容**：
- 2D 笔迹轨迹还原动画（按书写顺序逐笔画回放）
- 压力热力图叠加显示
- 各笔画评分标注

## 5. 系统模块划分

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **数据采集模块** | 驱动触觉传感器和相机，同步采集数据 | 用户书写 | 触觉数据流 + 成字照片 |

| **推理评估模块** | 加载模型，对新数据进行笔画识别和质量评分 | 触觉数据 + 成字照片 | 笔画类别 + 质量评分 |
| **可视化模块** | 笔迹还原动画、热力图、评分展示 | 触觉数据 + 字形轮廓 + 评分结果 | 可视化界面 |

## 6. 数据流总览

### 6.1 训练阶段数据流

```
用户书写
    ├── 触觉传感器 → 触觉数据流 → 抬笔/落笔分割 → 笔画片段[N]
    └── 顶部相机 → 成字照片 → OCR 字识别 → 字标签
                                    ↓
                          查笔画类别库 → 标准笔画序列
                                    ↓
                          笔画分割 → 笔画轮廓[N]
                                    ↓
                          视觉质量评估 → 字形质量标签
                                    ↓
                          时序对齐（按书写顺序匹配）
                                    ↓
                    训练数据集：[触觉片段, 笔画类别, 质量标签, 字形轮廓]
                                    ↓
                          Transformer 模型训练
```

### 6.2 推理阶段数据流

```
用户书写
    ├── 触觉传感器 → 触觉数据流 → 笔画分割 → 笔画片段[N]
    └── 顶部相机 → 成字照片 → 笔画分割 → 字形轮廓[N]
                                    ↓
                    笔画片段 → 模型推理 → 笔画类别 + 质量评分
                    字形轮廓 + 触觉方向数据 → 笔迹还原动画
                                    ↓
                          可视化展示界面
```

## 8. 非功能性需求

- **评分准确性**：笔画识别准确率目标 ≥ 90%，质量评分与专家评分相关性 ≥ 0.8
- **处理延迟**：离线批处理模式，单字完整分析时间 ≤ 5 秒
- **可扩展性**：模型架构支持后续添加新笔画类别、新评分维度
- **数据存储**：支持触觉数据和标注数据的持久化存储与管理

## 10. 术语表

| 术语 | 定义 |
|------|------|
| 触觉数据 | 智能毛笔笔尖传感器采集的多维信号（力度 + 方向 2D 向量，共 3 维） |
| 笔画片段 | 一次落笔到抬笔之间的连续触觉数据段 |
| 笔画类别库 | 存储笔画分类体系（~29 类）和汉字-笔画序列映射关系的本地数据库 |
| 标准笔画库 | 书法专家书写的触觉数据集合，作为质量评分的参考基准 |
| 视觉质量评估 | 利用成字照片自动评估字形质量的方法（传统图像处理 / 深度学习 / 混合方案） |
| 笔迹还原 | 仅凭触觉数据 + 字形轮廓，在屏幕上重建书写过程的动画 |
| Active Learning | 一种迭代训练策略，优先标注模型最不确定的样本以提升标注效率 |
